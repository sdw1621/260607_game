## 참고
- 계획서: `C:\Users\user\Desktop\260607_game\apps\2048\spec.md` 를 먼저 읽고 그대로 구현한다.

## 제약
- HTML, CSS, JavaScript만 사용. 외부 라이브러리 사용 금지(빌드 도구/프레임워크 금지). 순수 바닐라.
- 웹앱은 /apps/2048/ 안에서 자체 완결되어야 한다.
- 모바일에서도 동작해야 한다(터치 스와이프).

## 구현 요구사항
1. **보드**: 4x4 격자. 시작 시 타일 2개 생성(2: 90%, 4: 10%).
2. **이동/병합**: 방향키(←↑→↓)로 이동. 같은 숫자 병합, 한 이동당 한 타일 1회만 병합. 보드가 변했을 때만 새 타일 1개 추가.
3. **점수판**: 현재 점수(병합값 누적) + 최고 점수. 최고 점수는 localStorage 키 `2048-best`에 저장하여 새로고침/리셋해도 유지.
4. **승리/패배**: 2048 도달 시 승리 오버레이(계속하기 가능). 빈 칸 없고 이동 불가 시 게임 오버 오버레이.
5. **새 게임 버튼**: 보드/현재 점수 초기화(최고 점수는 유지).
6. **모바일**: 터치 스와이프(임계값 약 30px), `touch-action: none`, viewport 메타 태그로 줌/스크롤 방지.
7. **UI**: 원작 톤(배경 #faf8ef, 보드 #bbada0, 타일 값별 색상 단계). 타일 등장/병합 시 가벼운 애니메이션.
