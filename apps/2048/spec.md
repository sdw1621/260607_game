# 2048 게임 구현 계획 (spec.md)

## 1. 게임 개요

2048은 4x4 격자에서 진행되는 싱글 플레이어 슬라이딩 퍼즐 게임이다. 플레이어는 방향키(또는 터치 스와이프)로 모든 타일을 한 방향으로 민다. 같은 숫자를 가진 두 타일이 충돌하면 하나로 합쳐지며 숫자가 두 배가 된다. 목표는 2048 타일을 만드는 것이며, 더 이상 움직일 수 없으면 게임 오버다.

### 핵심 규칙

- **격자**: 4x4 (총 16칸). 빈 칸은 값 0 또는 null로 표현한다.
- **시작 상태**: 게임 시작 시 무작위 빈 칸 2개에 타일을 생성한다. 각 타일은 90% 확률로 2, 10% 확률로 4.
- **타일 이동**: 방향키를 누르면 해당 방향으로 모든 타일이 끝까지 미끄러진다. 이동 가능한 칸이 있으면 빈 칸을 메우며 붙는다.
- **타일 병합**:
  - 같은 숫자 타일 두 개가 이동 방향으로 인접해 충돌하면 하나로 합쳐지고 값은 두 배가 된다.
  - 한 번의 이동에서 하나의 타일은 한 번만 병합된다. (예: 2,2,2,2를 왼쪽으로 밀면 4,4가 됨. 8이 아님.)
  - 병합 처리 순서는 이동 방향 기준 가장 먼저 도달하는 타일부터다.
- **새 타일 생성**: 이동/병합으로 보드 상태가 실제로 변했을 때만, 이동 후 무작위 빈 칸 1개에 새 타일(2 또는 4)을 생성한다. 보드가 변하지 않았으면 새 타일을 생성하지 않는다.
- **점수**: 병합이 일어날 때마다 병합 결과 값(합쳐진 새 숫자)만큼 현재 점수에 더한다.
- **승리 조건**: 보드에 2048 타일이 처음 만들어지면 승리 메시지를 표시한다. 사용자는 "계속하기"를 선택해 더 높은 점수에 도전할 수 있다(승리 메시지는 게임당 1회만 노출).
- **패배(게임 오버) 조건**: 빈 칸이 없고, 상하좌우 어느 방향으로도 병합 가능한 인접 타일이 없을 때 게임 오버를 표시한다.

## 2. 파일 구조

웹앱은 `/apps/2048/` 폴더에 독립 완결된다. 블로그의 다른 파일에 의존하지 않는다.

```
/apps/2048/
├── index.html      # 게임 마크업: 헤더, 점수판, 보드 격자, 버튼, 메시지 오버레이
├── style.css       # 레이아웃, 색상, 타일 스타일, 반응형, 애니메이션
├── script.js       # 게임 로직: 상태, 이동/병합, 입력 처리, 렌더링, localStorage
└── spec.md         # 본 계획서
```

외부 라이브러리는 사용하지 않는다(순수 HTML/CSS/JS). 필요 시 CDN 폰트만 허용하나, 시스템 폰트 스택으로 충분하므로 기본은 외부 의존성 0.

## 3. 주요 기능 목록

### 3.1 입력 처리
- **키보드**: `keydown` 이벤트로 ArrowUp/Down/Left/Right 감지. 기본 스크롤 동작 방지(preventDefault). WASD 보조 지원(선택).
- **터치 스와이프(모바일)**: `touchstart`/`touchend`로 시작-끝 좌표 차이를 계산. 임계값(예: 30px) 이상이고 가로/세로 중 변위가 큰 축을 이동 방향으로 결정. 스크롤/줌 방지를 위해 보드 영역 `touch-action: none` 적용.

### 3.2 이동/병합 로직
- 보드를 2차원 배열(4x4)로 관리.
- 이동 방향에 관계없이 단일 "왼쪽으로 밀기" 함수로 일반화: 각 행을 추출 → 회전/전치로 방향 정규화 → 압축(빈 칸 제거) → 병합 → 다시 압축 → 역변환. 
  - 위/아래/오른쪽 이동은 보드를 전치(transpose)하거나 행을 뒤집은(reverse) 뒤 동일 로직 적용 후 복원.
- `moved` 플래그로 보드 변화 여부를 판정해 새 타일 생성 여부와 입력 무시를 결정.

### 3.3 점수판
- **현재 점수(score)**: 병합 시 누적. 화면 상단 점수 박스에 표시.
- **최고 점수(best)**: `localStorage` 키 `2048-best`에 저장/로드. 현재 점수가 최고 점수를 넘으면 즉시 갱신 및 저장.
- 새 게임으로 리셋해도 최고 점수는 유지된다.

### 3.4 게임 오버 / 승리 처리
- 매 이동 후 승리(2048 등장) 및 패배(이동 불가) 여부 검사.
- 결과를 반투명 오버레이 + 메시지("You Win!" / "Game Over")로 표시.
- 오버레이에 "다시 시작" 버튼(패배) 또는 "계속하기"/"다시 시작"(승리) 버튼 제공.

### 3.5 새 게임(리셋) 버튼
- 헤더의 "새 게임" 버튼 클릭 시 보드 초기화, 현재 점수 0, 시작 타일 2개 재생성. 최고 점수는 유지.

## 4. 모바일 지원 방안

- **반응형 레이아웃**: 보드 크기를 뷰포트 기반(`min(90vw, 460px)` 등)으로 설정해 작은 화면에서도 정사각형 유지. CSS Grid 또는 비율 기반 패딩으로 정사각 셀 유지.
- **터치 스와이프**: 위 3.1 참조. 4방향 스와이프로 키보드 화살표와 동일하게 동작.
- **스크롤/줌 방지**: 보드 컨테이너에 `touch-action: none`, viewport 메타(`width=device-width, initial-scale=1, user-scalable=no`) 설정.
- **터치 타깃**: 버튼 최소 44px 높이 확보.

## 5. UI/UX 개요

### 레이아웃 (위에서 아래로)
1. **헤더**: 좌측 "2048" 타이틀, 우측 점수판(현재 점수 박스 + 최고 점수 박스).
2. **서브 헤더**: 간단한 안내 문구 + "새 게임" 버튼.
3. **보드**: 4x4 격자. 배경 셀(빈 칸) 위에 타일이 올라가는 구조. 중앙 정렬.
4. **하단**: 조작 방법 안내(화살표/스와이프).
5. **오버레이**: 승리/패배 시 보드 위에 표시.

### 색상 (원작 톤 기반)
- 배경: 따뜻한 베이지 `#faf8ef`
- 보드 배경: `#bbada0`
- 빈 셀: `#cdc1b4`
- 타일 값별 색상 단계: 2(#eee4da), 4(#ede0c8), 8(#f2b179), 16(#f59563), 32(#f67c5f), 64(#f65e3b), 128~2048(#edcf72 → #edc22e 계열, 점진적으로 진해짐).
- 타일 글자색: 2,4는 어두운 색(#776e65), 8 이상은 흰색(#f9f6f2).
- 큰 숫자(128 이상)는 폰트 크기 자동 축소.

### 인터랙션/애니메이션
- 타일 등장 시 살짝 팝(scale) 애니메이션, 병합 시 강조 펄스.
- CSS transition으로 부드러운 느낌(과도한 라이브러리 없이 transform/opacity 위주).
- 접근성: 점수판에 `aria-live`로 점수 변경 안내(선택).

## 6. 구현 단계 목록

1. **HTML 골격(index.html)**: 헤더/점수판/버튼/보드 컨테이너/오버레이 마크업 작성. viewport 메타 포함.
2. **CSS 레이아웃(style.css)**: 보드 그리드, 정사각 셀, 색상 팔레트, 점수판/버튼 스타일, 반응형, 기본 애니메이션.
3. **상태 모델(script.js)**: 4x4 보드 배열, score/best 변수, 초기화 함수, 무작위 타일 생성 함수.
4. **렌더링**: 보드 배열 → DOM 갱신 함수. 점수판 갱신 함수.
5. **이동/병합 엔진**: 왼쪽 밀기 일반화 함수 + 방향별 변환(transpose/reverse), moved 판정, 병합 시 점수 누적.
6. **입력 연결**: 키보드 이벤트 + 이동→새 타일 생성→렌더 파이프라인.
7. **승리/패배 판정**: 2048 검사, 이동 가능 여부 검사, 오버레이 표시/버튼 처리.
8. **점수 영속화**: localStorage로 best 로드/저장.
9. **모바일 터치**: 스와이프 제스처 → 이동 파이프라인 연결, touch-action 처리.
10. **새 게임 버튼 + 폴리시**: 리셋 동작, 승리 후 계속하기 처리.
11. **점검**: 다양한 화면 크기/키보드/터치 동작 확인(브라우저 수동 테스트는 Review 단계에서 수행).

## 7. 검증 포인트(Review 단계 참고)

- 2,2,2,2 왼쪽 밀기 → 4,4 (8 아님) 확인.
- 보드 변화 없을 때 새 타일 미생성 확인.
- best 점수가 리셋/새로고침 후에도 유지되는지 확인.
- 모바일 스와이프 4방향 정상 동작 및 스크롤 방지 확인.
- 게임 오버 판정이 빈 칸 없음 + 병합 불가 동시 충족 시에만 발동하는지 확인.
