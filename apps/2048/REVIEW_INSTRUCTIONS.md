# Review 서브에이전트 지침 — 2048 게임

## 역할
너는 Review 서브에이전트다. Build 서브에이전트가 만든 2048 게임을 검증한다. Build와 분리된 독립 검증자다.

## 검증 대상 파일
- `C:\Users\user\Desktop\260607_game\apps\2048\index.html`
- `C:\Users\user\Desktop\260607_game\apps\2048\style.css`
- `C:\Users\user\Desktop\260607_game\apps\2048\script.js`

계획서 `spec.md` 기준으로 요구사항이 충족되었는지 확인한다.

## 검증 항목
1. **브라우저 동작**: Preview MCP(또는 가능한 방법)로 실제 로드하여 콘솔 에러가 없는지, 화면이 정상 렌더링되는지 확인한다.
2. **이동/병합 로직 정확성**: 코드를 정독하여 다음을 검증한다.
   - 한 이동당 한 타일이 1회만 병합되는가 (2,2,2,2 → 4,4)
   - 보드가 변했을 때만 새 타일이 추가되는가
   - 점수가 병합값만큼만 정확히 증가하는가
3. **점수판**: 현재 점수 + 최고 점수(localStorage `2048-best`) 저장/유지 동작.
4. **승리/패배 판정**: 2048 도달 시 승리, 이동 불가 시 게임 오버.
5. **새 게임 버튼**: 보드/현재 점수 초기화, 최고 점수 유지.
6. **모바일**: 터치 스와이프, viewport/touch-action 설정.
7. **자체 완결성**: 외부 라이브러리 의존 없는지, /apps/2048/ 안에서 완결되는지.

## 수정 범위
- 명백한 버그(동작 불가, 콘솔 에러, 로직 오류)를 발견하면 위 3개 파일에 한해 직접 수정한다.
- 단순 취향 차이는 수정하지 말고 review.md에 기록만 한다.

  - 최종 판정(통과/조건부 통과/실패)
